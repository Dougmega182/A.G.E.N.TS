# Security Policy

Document ID: AGENTS-SEC-001
Status: Draft
Version: 1.0.0
Classification: RESTRICTED
Owner: Security Command
Approval: Audit Agent + Gatekeeper

Purpose
Define security controls aligned to PSPF, ISM, and Essential Eight.

Control Domains
1. Governance and accountability
- Security ownership defined and audited quarterly.
2. Access control
- Least privilege. Strong authentication. Service identities for agents.
3. Logging and monitoring
- All access and decisions logged and monitored.
4. Incident response
- RED and BLUE protocols govern security incidents.
5. Data protection
- Encryption for CONF and TS data. Key rotation every 90 days.
6. Secure configuration
- Baseline hardened configs for all systems.
7. Vulnerability management
- Monthly assessments and remediation tracking.
8. Backup and recovery
- Daily backups and tested recovery procedures.

Mapping
- PSPF: Governance, personnel security, information security.
- ISM: Access control, logging, incident response, encryption.
- Essential Eight: Application control, patching, macro controls, admin privilege management, backups.
