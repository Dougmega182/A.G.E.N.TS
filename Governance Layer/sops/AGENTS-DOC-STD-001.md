## 1. Purpose and Scope

This SOP defines the required documentation standard for all governance, policy, SOP, and contract documents within Little Sunraes Pty Ltd and the A.G.E.N.T.S. system. It applies to all documents created, updated, or approved under the corporate governance framework.

---

## 2. Mandatory Document Structure

All documents must include, in this order:

1. Title
2. Document metadata table (Document ID, Version, Status, Classification, Owner, Authority, Review Cycle)
3. Purpose and Scope
4. Application (where relevant)
5. Legislative and Policy Framework (where relevant)
6. Requirements or Procedures
7. Records Management and Retention
8. Approval and Authorisation
9. Document Control (Version history)

---

## 3. Document Identification

- Document IDs must use the format: `AGENTS-<AREA>-<NNN>` or `HR-EMP-001` style for employment documents.
- Schedules and annexes must use the parent document ID with suffix (e.g., `HR-EMP-001-S1`).
- File names must include the Document ID as the primary identifier.

---

## 4. Classification

Classification must be one of:
- INTERNAL
- RESTRICTED
- CONFIDENTIAL
- TOP_SECRET

Classification must align to the Information Sharing Rules.

---

## 5. Language and Style

- Formal corporate tone (government-grade).
- Avoid ambiguous or conversational language.
- Use clear section numbering and headings.
- Mandatory statements use "must" or "shall".

---

## 6. Version Control

- Every document must include a Document Control table.
- Updates require a new version number and change description.
- Draft documents must not be treated as approved.

---

## 7. Approval Workflow

- All documents must follow the Approval Workflow (AGENTS-APPROVAL-001).
- Approval records must be stored under `Governance Layer/records/approvals/`.

---

## 8. Storage Location

- Governance documents: `Governance Layer/`
- Policies: `Governance Layer/policies/`
- Onboarding documents: `Onboarding/`
- Approval records: `Governance Layer/records/approvals/`

---

## 9. Compliance

Non-compliant documents must be revised before approval. Failure to adhere to this SOP is a governance breach.
