STAGE 04

PROTOCOL FRAMEWORK

"What happens when things go wrong. Every failure mode has a named response."
RED-1
TRIGGER: Critical error, logic failure, or data inconsistency in any domain
Mandatory rollback to Planning Layer. All downstream production in the affected domain suspended.
  • Audit Agent flags with domain + context reference
  • Immediate thread suspension in affected domain
  • Root Cause Analysis filed within 4 hours
  • Correction session with domain-specific agents
  • Re-audit before resumption
  • 3× RED-1 in one domain = mandatory Structural Review
BLUE-1
TRIGGER: Legal, financial, or compliance risk in any domain
Immediate production halt. Compliance Agent notified. All flagged outputs sequestered.
  • Compliance Agent issues flag with source reference
  • No further distribution of flagged output
  • Risk assessment within 2 hours
  • Remediation brief issued to responsible agent
  • Two independent audit passes before clearance
GOLD-1
TRIGGER: Output fails domain-specific quality standards
Return to Execution Design Layer. QA Agent issues remediation brief. Standards must be met before resubmission.
  • QA Agent documents specific failure criteria
  • Affected output rejected — not revised in place
  • Return to Layer 4 with remediation brief
  • Resubmission undergoes full Layer 5 validation
  • Two GOLD-1 failures on same output = Planning Layer review
BLACK-1
TRIGGER: Simultaneous RED-1 + BLUE-1, or Constitutional Violation
Full system halt. All production across ALL domains ceases. Constitutional Triumvirate convenes within 24 hours.
  • All agents notified immediately
  • Triumvirate convenes within 24 hours
  • Full Pipeline Integrity Audit across all domains
  • Written Recovery Plan required before resumption
  • Gatekeeper must personally approve recovery plan
AMBER-1
TRIGGER: ADHD overwhelm detected — task queue overload, decision paralysis, or executive dysfunction indicators
Automatic task triage. Reduce visible queue to single next action. Defer all non-critical decisions. Activate recovery scaffolding.
  • Executive Function Agent triggers immediately
  • All domains except current priority are paused
  • Task queue reduced to maximum 3 visible items
  • Single "Next Action" surfaced with clear instructions
  • Dopamine-reward task interspersed if available
  • Duration: until Gatekeeper signals recovery
GREEN-1
TRIGGER: Flow state / momentum detected — Gatekeeper is producing at high velocity
Protect flow state. All non-emergency notifications deferred. Only RED-1 or BLACK-1 may interrupt.
  • Executive Function Agent activates shield
  • All lower-priority interrupts queued silently
  • Cross-domain task requests held until flow ends
  • Session duration logged for pattern analysis
  • Natural break point suggested (not forced) at 90-min intervals
WHITE-1
TRIGGER: Cross-domain resource conflict — two or more domains competing for Gatekeeper attention/time
Boardroom arbitration required. Northern Star priority order governs. Executive Function Agent presents options.
  • Competing domain directors present cases
  • Board votes on priority allocation
  • Northern Star priority order is the tiebreaker
  • Decision logged with reasoning
  • Losing domain receives rescheduled timeline

        task = create_task(
            origin=result.auditor,

            type="correction",

        )

        log_event("TASK_AUTO_CREATED", task)

        # ADHD: check if new task pushes overwhelm threshold
        if gatekeeper.active_tasks > OVERWHELM_THRESHOLD:
            trigger_protocol("AMBER-1")

def route_to_least_loaded(skill):
    eligible = [a for a in agents if skill in a.skills]
    return min(eligible, key=lambda a: a.current_tasks)
